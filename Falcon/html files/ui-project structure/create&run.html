Step 4 : Setting up Test Environment in config.properties:

Configure Test Environment under config.properties as below:
Important Configurations:
  • i.testreport.uploadurl– Report Dashboard Upload URL (if uploadResults=true)
  • ii.onfailureimage.uploadurl– Report Dashboard Image Upload URL (if uploadResults=true & capture.screenshot=true)

  • iii.instances– Number of Parallel Test Runs (1 for Single Run, 2 for Parallel Run)
  • iv.ClientName– Client Name to identify Reports under Report Dashboard
  • v.ModuleName– This is the suite files separated by comma (,) without any extension.

    • For Example: demoSuite

  • vi.BrowserCaps– Browser and OS Details, Syntax will be OSName_OSVersion_BrowserName_BrowserVersion We can ‘ve multiple BrowserCaps for Parallel run when instances are more than 1.

    • For Example:
    BrowserCaps=WIN_10_chrome_66 (when instances as 1).
    BrowserCaps=WIN_10_chrome_66,WIN_10_ie_11 (when instances as 2).

  • vii.Environment/gridIp – This will work as below:
    • Local – To Run in the Local Machine.
    Grid – To Run in the Grid Machine (gridIP should be mentioned).
    Browserstack – To Run in the Cloud Machine (Browser Stack).

  • viii.automatic.bug.creation – To Create JIRA Issues if there is an error.
  • xi.capture.screenshot – To Capture Screenshot if there is an error.
  • x.uploadResults – To Upload Results to Report Dashboard.
